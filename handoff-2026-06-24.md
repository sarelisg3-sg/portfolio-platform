## Decisiones importantes

### Arquitectura
- `portfolio-platform` es el hub central — no código, solo docs, PRD, ADRs, roadmap e issues.
- No monorepo: cada proyecto es un repo independiente bajo `sarelisg3-sg/`.
- GitHub-first deploy para todos los proyectos nuevos: `git push → CI → Vercel auto-deploy` (ver ADR-002). El patrón `npx vercel deploy --yes` queda obsoleto para proyectos nuevos.

### Deploy de proyectos existentes
- `le-point-du-fle-next` (en `sarelisg3-sg/le-point-du-fle`): sigue con deploy CLI por ahora. Migrar a GitHub-first cuando haya un cambio significativo. No bloquea P01.
- `todo-app`: repo solo local, no es case study prioritario.

### Dependencias críticas bloqueantes
- **P08** no puede arrancar sin P01 terminado (productiza el design system).
- **P14** no puede arrancar sin P11 terminado (evolución del UX Reviewer a SaaS).
- **P16** no puede arrancar sin P01 + P08 + P13 terminados (docs inteligentes del design system con RAG).
- **P13** necesita P02 terminado (case studies MDX = fuente del RAG).

### Testing seams (ver ADR-003)
1. `lib/` utilities → Vitest unit tests (lógica pura separada de proveedores)
2. Components → Storybook + axe-core + Testing Library para interactivos
3. API Routes → integration tests contra route handlers reales (no mockear lógica principal)
4. npm package → CI gate antes de publish (pertenece a P08, no P01)

### Separación P01 / P08
Esta distinción es crítica. P01 construye para uso propio. P08 construye para distribución.
**No pertenece a P01:** exports field en package.json, peerDependencies, build con tsup/rollup, barrel export `index.ts`, npm publish workflow, snapshot tests, versionado semántico, CHANGELOG.
