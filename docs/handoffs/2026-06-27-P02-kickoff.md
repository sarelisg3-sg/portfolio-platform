## Pasos recomendados para la sesión

### 1. Crear issue de kickoff en portfolio-platform
```bash
gh issue create \
  --repo sarelisg3-sg/portfolio-platform \
  --title "[Fase 1 · P02] Portfolio V1" \
  --body "Kickoff de Portfolio V1 — sitio profesional con Next.js 16, Tailwind v4, MDX, Framer Motion y deploy en Vercel."
```

### 2. Crear repo portfolio-v1
```bash
gh repo create sarelisg3-sg/portfolio-v1 --public --description "Portfolio profesional V1 — Next.js 16, Tailwind v4, MDX, Framer Motion"
```

### 3. Inicializar Next.js
```bash
npx create-next-app@latest portfolio-v1 \
  --typescript --tailwind --eslint --app --src-dir=no --import-alias="@/*"
```
Luego revisar `node_modules/next/dist/docs/` antes de escribir cualquier código (regla del AGENTS.md).

### 4. Instalar dependencias del stack P02
```bash
npm install framer-motion @next/mdx next-mdx-remote
npm install resend
npx shadcn@latest init
```

### 5. Conectar a Vercel y primer deploy
```bash
vercel --prod
```

### 6. Actualizar projects.json
Cambiar `portfolio-v1` de `not-started` a `in-progress` y agregar `repo`, `deployment`, `kickoff_issue`.
